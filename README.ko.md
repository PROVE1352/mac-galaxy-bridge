# Mac–Galaxy Bridge

> 맥 + 갤럭시 조합에서 애플 Continuity 경험을 — 탭 0회 파일 전송과 원클릭 인스턴트 핫스팟.

[English README](README.md)

## 왜 만드나

맥과 갤럭시를 같이 쓰면 Continuity 없이 살게 된다:

- **파일 전송**: 최신 갤럭시/픽셀은 AirDrop 상호운용이 되지만, 폰에서 받을 때마다 공개 범위를 "모든 사람"으로 켜고 수락을 눌러야 한다. 서드파티 앱은 양쪽에서 앱을 열어야 하고.
- **핫스팟**: 애플 인스턴트 핫스팟(맥이 아이폰 핫스팟을 원격으로 켬, 비번 불필요)의 안드로이드 대응물이 없다. 폰 꺼내고, 설정 뒤지고, 토글 켜고, 기다린다.

이 프로젝트는 **내 기기 3대** — 맥북, 갤럭시 S25, 갤럭시 탭 S11 울트라 — 에서 이 두 간극을 없앤다.

## 뭘 하나

| 기능 | UX |
|---|---|
| **탭 0회 파일 전송** | 폰/탭 → 맥: 공유 시트 → "내 Mac" → 끝 (`~/Downloads` 자동 저장). 맥 → 폰/탭: Finder 우클릭 → "S25로 보내기" → 끝. 공개 범위 토글도, 수락 탭도 없음. |
| **원클릭 핫스팟** | 맥 메뉴바에서 "연결" 클릭 → 폰이 알아서 핫스팟을 켬 → 맥 자동 조인. 폰은 주머니 속에. |
| **태블릿 지원** | 같은 컴패니언 앱이 탭 S11 울트라에서도 돈다. |

## 어떻게 동작하나

수락 탭 마찰은 퀵쉐어/에어드랍이 **모르는 기기**를 다루는 신뢰 모델에 내장된 것이라 그 프로토콜 위에서는 제거가 불가능하다. 그래서 페어링된 기기끼리는 자체 프로토콜로 말한다: mDNS 발견, 최초 1회 코드 페어링(맥이 8자 코드를 표시, 폰에서 입력 — HMAC이 양쪽 TLS 지문을 묶어 중간자를 차단), 인증서 피닝된 TLS, 페어링 기기 간 무조건 자동 수락 (KDE Connect의 신뢰 모델을 이 기기 3대에 맞춤). 상세는 [프로토콜 명세](docs/PROTOCOL.md).

핫스팟은 폰 쪽에 특권 API가 필요 없다: 맥이 블루투스 연결을 트리거하면 컴패니언 앱이 `ACL_CONNECTED` 브로드캐스트를 받아 접근성 서비스로 설정의 핫스팟 스위치를 직접 누른다. (코드 없는 두 경로 — 셸 테더링 API, 삼성 루틴 — 는 모두 막혀 있음이 [M1 검증](docs/M1_HOTSPOT_FINDINGS.md)에서 확인됐다.)

아키텍처 다이어그램은 [영문 README](README.md#how-it-works) 참고.

## 로드맵

- [x] **M1 — 원클릭 핫스팟 (검증)**: 코드 없는 두 경로 모두 Android 16에서 차단 확인 → 핫스팟은 컴패니언 앱으로 편입 ([결과](docs/M1_HOTSPOT_FINDINGS.md))
- [x] **M2 — 전송 코어 + 핫스팟 토글**: 컴패니언 앱(mDNS + 코드 페어링 + TLS 피닝, 자동 수락 수신) + 맥 메뉴바 앱(Bonjour, 피닝 TLS, 송수신, 원클릭 핫스팟)
- [x] **M3 — OS 통합**: Finder 빠른 동작("Send to Galaxy"), 안드로이드 공유시트 타깃, 태블릿 롤아웃
- [x] **M4 — 부가기능**: 클립보드 푸시(맥 → 폰), 전송 기록

## 상태

M2–M4 구현 완료 — 실기기 검증은 [docs/TESTING.md](docs/TESTING.md) 체크리스트로 진행. 일단 개인 3대 기기용으로 만들고, 일반화는 나중에.
